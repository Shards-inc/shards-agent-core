# Contributing to Shards AI Agent System

We welcome contributions to the Shards AI Agent System! By participating in this project, you agree to abide by our [Code of Conduct](CODE_OF_CONDUCT.md).

## How to Contribute

### 1. Fork the Repository
Fork the `shards-agent-core` repository to your GitHub account.

### 2. Clone Your Fork
```bash
git clone https://github.com/YOUR_USERNAME/shards-agent-core.git
cd shards-agent-core
```

### 3. Create a New Branch
Create a new branch for your feature or bug fix:
```bash
git checkout -b feature/your-feature-name
# or
git checkout -b bugfix/issue-description
```

### 4. Make Your Changes
Implement your changes, ensuring they adhere to the project's coding standards and architecture. Add unit tests for new features or bug fixes.

### 5. Test Your Changes
Run existing tests and any new tests you've added:
```bash
python -m unittest discover tests
```

### 6. Commit Your Changes
Commit your changes with a clear and concise commit message. Follow the Conventional Commits specification (e.g., `feat: add new agent type`, `fix: resolve bug in workflow orchestration`).

### 7. Push to Your Fork
```bash
git push origin feature/your-feature-name
```

### 8. Create a Pull Request
Open a pull request from your fork to the `main` branch of the `Shards-inc/shards-agent-core` repository. Provide a detailed description of your changes and reference any related issues.

## Coding Standards
- Follow PEP 8 for Python code.
- Use type hints for all function arguments and return values.
- Write clear and concise comments where necessary.

## Issue Reporting
If you find a bug or have a feature request, please open an issue on the GitHub repository. Provide as much detail as possible, including steps to reproduce the issue or a clear description of the desired feature.

## Contact
For any questions or further information, please refer to the project maintainers or the Shards-inc organization on GitHub.
